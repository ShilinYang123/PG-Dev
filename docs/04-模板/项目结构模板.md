# 项目结构模板

## 概述

本文档提供了不同类型项目的标准目录结构模板，帮助开发团队快速建立规范、可维护的项目架构。所有模板都遵循业界最佳实践，支持团队协作和项目扩展。

## 最佳实践

### 目录命名规范

1. **使用小写字母和连字符**
   ```
   ✅ user-service
   ✅ api-client
   ❌ UserService
   ❌ apiClient
   ```

2. **复数形式用于集合**
   ```
   ✅ components/
   ✅ services/
   ✅ utils/
   ❌ component/
   ```

3. **功能导向的分组**
   ```
   ✅ features/auth/
   ✅ features/dashboard/
   ❌ pages/auth/
   ```

### 文件命名规范

1. **组件文件使用PascalCase**
   ```
   ✅ UserProfile.tsx
   ✅ ProductCard.vue
   ❌ userProfile.tsx
   ```

2. **工具文件使用camelCase**
   ```
   ✅ apiClient.ts
   ✅ dateHelpers.ts
   ❌ api-client.ts
   ```

3. **配置文件使用kebab-case**
   ```
   ✅ webpack.config.js
   ✅ docker-compose.yml
   ❌ webpackConfig.js
   ```

### 项目组织原则

1. **按功能分组，而非技术类型**
   ```
   ✅ features/user-management/
       ├── components/
       ├── services/
       └── types/
   
   ❌ components/user/
       services/user/
       types/user/
   ```

2. **保持浅层目录结构**
   - 避免超过3-4层的嵌套
   - 使用index文件简化导入

3. **分离关注点**
   - 业务逻辑与UI分离
   - 配置与代码分离
   - 测试与源码分离

4. **可预测的结构**
   - 统一的命名约定
   - 一致的目录组织
   - 清晰的职责划分

### 扩展性考虑

1. **模块化设计**
   - 每个模块独立可测试
